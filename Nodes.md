# SampleChain 节点角色与关系说明

## 节点角色

你的代码里实际上只有一种节点类型 `StartServer`，但根据启动参数的不同，节点会表现出不同的行为：

- **种子节点（Seed Node）** — 硬编码为 `localhost:3000`，是整个网络的"入口"。其他节点启动时都会先联系它，它负责在节点之间转发交易 inv 消息，但自己不挖矿（`mineAddress` 为空）。

- **普通节点** — 端口非 3000，`mineAddress` 为空。只负责同步区块链数据，不参与挖矿。

- **矿工节点** — 端口非 3000，启动时传入了 `minerAddress`。在收到足够交易后会自动打包挖矿。

---

## 启动流程

```
矿工节点/普通节点启动
        │
        ▼
  SendVersion(种子节点)          ← 发送自己的链高度
        │
        ▼
种子节点 HandleVersion
  比较高度，若对方落后
        │
        ▼
  SendGetBlocks(新节点)          ← 向新节点要区块哈希列表
        │
        ▼
新节点 HandleGetBlocks
        │
        ▼
  SendInv(种子节点, "block", 所有哈希)
        │
        ▼
种子节点逐块 SendGetData / HandleBlock
        └─ 同步完成后 UTXOSet.Reindex()
```

---

## 交易广播流程

```
用户发起转账
        │
        ▼
  SendTx(种子节点, tx)
        │
        ▼
种子节点 HandleTx
  nodeAddress == KnownNodes[0]  ← 种子节点判断逻辑
        │
        ▼
  for 每个已知节点:
    SendInv(node, "tx", tx.ID)  ← 只发哈希，不发完整交易
        │
        ▼
矿工节点 HandleInv
        │
        ▼
  SendGetData(种子节点, "tx", txID)  ← 用哈希换完整交易
        │
        ▼
  memoryPool[txID] = tx
  if len(memoryPool) >= 2 && mineAddress != ""
        │
        ▼
      MineTx()
```

> **关键设计**：种子节点只转发交易 ID（inv），不直接转发完整交易内容，矿工节点收到 ID 后再主动来取，这样避免了大量数据的无效广播。

---

## 挖矿流程

```
MineTx()
    │
    ├─ 验证内存池所有交易签名
    ├─ 附加 coinbase 交易（矿工奖励）
    ├─ CreateBlock() → PoW 挖矿
    ├─ UTXOSet.Reindex()         ← 重建 UTXO 索引
    ├─ 清空已打包交易的内存池
    ├─ SendInv(所有节点, "block", 新区块哈希)  ← 广播新区块
    └─ if memoryPool 还有剩余 → 递归 MineTx()
```

挖矿完成后，矿工节点向所有已知节点广播新区块哈希，其他节点收到后再用 `GetData` 拉取完整区块，最终所有节点同步到同一条链。

---

## 三者关系总结

```
        用户
         │  发交易
         ▼
     种子节点 :3000
     ┌────────────────────────────┐
     │ · 维护 KnownNodes 列表      │
     │ · 转发交易 inv 给所有节点    │
     │ · 不挖矿                    │
     └──────┬─────────────────────┘
            │ SendInv("tx")
     ┌──────▼──────────┐
     │   矿工节点        │
     │ · 拉取完整交易    │
     │ · 积累≥2笔后挖矿  │
     │ · 广播新区块      │
     └──────┬───────────┘
            │ SendInv("block")
     ┌──────▼──────────┐
     │   普通节点        │
     │ · 同步区块数据    │
     │ · 维护本地链副本  │
     └─────────────────┘
```

**一句话概括**：种子节点是消息中转站，矿工节点是生产者，普通节点是数据同步者。三者都运行相同的代码，角色完全由启动参数决定。
